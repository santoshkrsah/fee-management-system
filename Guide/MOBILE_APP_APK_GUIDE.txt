What You Need:

PART 2: GENERATE APP ICONS
═══════════════════════════════════════════════════════════════════════

You need icons in multiple sizes for different devices and purposes.

METHOD A: Use Online Icon Generator (Easiest)
───────────────────────────────────────────────────────────────────────

1. Go to: https://www.pwabuilder.com/imageGenerator

2. Upload your school logo (512x512px PNG recommended)

3. Click "Download" to get all icon sizes

4. Extract the zip file

5. Upload all icons to: /assets/images/ folder on your server

   Required icon sizes:
   - icon-72x72.png
   - icon-96x96.png
   - icon-128x128.png
   - icon-144x144.png
   - icon-152x152.png
   - icon-192x192.png
   - icon-384x384.png
   - icon-512x512.png

METHOD B: Use RealFaviconGenerator (Alternative)
───────────────────────────────────────────────────────────────────────

1. Go to: https://realfavicongenerator.net/

2. Upload your logo image

3. Customize settings for Android, iOS, etc.

4. Download the package

5. Upload icons to /assets/images/

METHOD C: Manual Creation (If you know Photoshop/GIMP)
───────────────────────────────────────────────────────────────────────

1. Open your logo in Photoshop/GIMP

2. Create these sizes manually:
   72x72, 96x96, 128x128, 144x144, 152x152, 192x192, 384x384, 512x512

3. Save as PNG with transparency

4. Upload to /assets/images/

PART 3: ADD PWA CODE TO YOUR WEBSITE
═══════════════════════════════════════════════════════════════════════

Step 1: Update Your PHP Files
───────────────────────────────────────────────────────────────────────

Add this line in the <head> section of your main pages:

For login.php, dashboard.php, and other main pages, add after the
existing <head> tags:

```php
<?php include '../includes/pwa_head.php'; ?>
```

Or for admin pages:
```php
<?php include __DIR__ . '/../includes/pwa_head.php'; ?>
```

Files to update:
□ /admin/login.php
□ /admin/dashboard.php
□ /includes/header.php (if you have a common header file)
□ /modules/*/list.php (student, fee collection pages)

Step 2: Upload PWA Files to Server
───────────────────────────────────────────────────────────────────────

Upload these files to your Hostinger public_html:

□ manifest.json (in root directory)
□ sw.js (service worker - in root directory)
□ includes/pwa_head.php
□ All icon files in /assets/images/

Step 3: Enable HTTPS (Critical!)
───────────────────────────────────────────────────────────────────────

1. Login to Hostinger hPanel
2. Go to Security → SSL
3. Enable SSL certificate (Free with Hostinger)
4. Force HTTPS redirect
5. Wait 5-10 minutes for SSL to activate

Your website should now be: https://yourdomain.com

PART 5: CREATE ANDROID APK - METHOD 1 (PWABuilder - Easiest)
═══════════════════════════════════════════════════════════════════════

This is the EASIEST method to convert your PWA to APK!

Step 1: Go to PWABuilder
───────────────────────────────────────────────────────────────────────

1. Open: https://www.pwabuilder.com/

2. Enter your website URL: https://yourdomain.com

3. Click "Start"

Step 2: Review Your PWA
───────────────────────────────────────────────────────────────────────

1. PWABuilder will analyze your website

2. You'll see scores for:
   - Manifest
   - Service Worker
   - Security

3. Fix any issues if shown (usually auto-detected)

Step 3: Generate APK
───────────────────────────────────────────────────────────────────────

1. Click on "Package For Stores" or "Android"

2. Choose "Google Play" or "Android Package"

3. Fill in app details:
   - Package ID: com.yourschool.feemgmt (use your school name)
   - App name: Fee Management System
   - Version: 1.0.0

4. Click "Generate"

5. Wait 2-5 minutes for APK generation

6. Download the APK file (free!)

Step 4: Install APK on Android
───────────────────────────────────────────────────────────────────────

1. Transfer the APK to your Android phone

2. Open the APK file

3. Allow "Install from unknown sources" if prompted

4. Install the app

5. Done! Your app is now installed!

PART 6: CREATE ANDROID APK - METHOD 2 (Trusted Web Activity)
═══════════════════════════════════════════════════────════════════════

More technical but gives you full control.

Requirements:
- Android Studio installed on computer
- Basic knowledge of Android development

Step 1: Install Bubblewrap
───────────────────────────────────────────────────────────────────────

1. Install Node.js from https://nodejs.org

2. Open terminal/command prompt

3. Run: npm install -g @bubblewrap/cli

Step 2: Initialize Project
───────────────────────────────────────────────────────────────────────

1. Create a new folder for your APK project

2. Open terminal in that folder

3. Run: bubblewrap init --manifest https://yourdomain.com/manifest.json

4. Answer the prompts:
   - Domain: yourdomain.com
   - Package name: com.yourschool.feemgmt
   - App name: Fee Management System

Step 3: Build APK
───────────────────────────────────────────────────────────────────────

1. Run: bubblewrap build

2. Wait for build to complete (5-10 minutes)

3. Find APK in: app/build/outputs/apk/release/

PART 7: CREATE ANDROID APK - METHOD 3 (Android Studio)
═══════════════════════════════════════════════════════════════────════

For advanced users who want maximum customization.

Step 1: Download Template
───────────────────────────────────────────────────────────────────────

1. Go to: https://github.com/GoogleChromeLabs/svgomg-twa

2. Download the TWA (Trusted Web Activity) template

3. Extract the zip file

Step 2: Configure Project
───────────────────────────────────────────────────────────────────────

1. Open Android Studio

2. Open the extracted project

3. Edit: app/build.gradle
   - Change package name
   - Update app name

4. Edit: app/src/main/res/values/strings.xml
   - Set your website URL
   - Set app name

5. Replace icons in: app/src/main/res/mipmap-*/

Step 3: Build Signed APK
───────────────────────────────────────────────────────────────────────

1. In Android Studio: Build → Generate Signed Bundle/APK

2. Choose APK

3. Create new keystore (save it securely!)

4. Build the APK

5. Find APK in: app/release/

PART 9: TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════

Issue 1: "Add to Home Screen" Not Showing
───────────────────────────────────────────────────────────────────────

Cause: PWA requirements not met

Fix:
□ Ensure HTTPS is enabled (SSL certificate)
□ Check manifest.json is accessible: https://yourdomain.com/manifest.json
□ Check service worker: https://yourdomain.com/sw.js
□ Verify all icon files exist
□ Try in Chrome Incognito mode
□ Clear browser cache

Issue 2: App Icons Not Showing
───────────────────────────────────────────────────────────────────────

Cause: Icon files missing or wrong path

Fix:
□ Verify icons exist at: /assets/images/icon-*.png
□ Check file names match manifest.json
□ Ensure PNG format, not JPG
□ Clear browser cache
□ Regenerate manifest with correct paths

Issue 3: Service Worker Error
───────────────────────────────────────────────────────────────────────

Cause: Service worker registration failed

Fix:
□ Check sw.js is in root directory
□ Check browser console for errors (F12 → Console)
□ Ensure HTTPS is enabled
□ Clear service worker: Chrome → DevTools → Application → Clear storage
□ Update service worker cache version

Issue 4: App Works in Browser But Not as APK
───────────────────────────────────────────────────────────────────────

Cause: APK configuration issue

Fix:
□ Check package name is correct
□ Verify start_url in manifest.json
□ Ensure domain ownership is verified
□ Rebuild APK with correct settings
□ Check Android version compatibility

Issue 5: Blank Screen After Install
───────────────────────────────────────────────────────────────────────

Cause: start_url incorrect or service worker cache issue

Fix:
□ Verify start_url in manifest.json points to valid page
□ Clear app data: Android Settings → Apps → Your App → Clear Data
□ Update service worker to cache correct pages
□ Ensure login page is cached

PART 11: UPDATING YOUR APP
═══════════════════════════════════════════════════════════════════════

When You Update Your Website:
───────────────────────────────────────────────────────────────────────

1. Make changes to your website code

2. Update service worker cache version in sw.js:
   Change: const CACHE_NAME = 'fee-management-v1.0';
   To:     const CACHE_NAME = 'fee-management-v1.1';

3. Upload updated files to server

4. Users will get update automatically within 24 hours

5. Or they can clear app data to force update

For Major Updates:
───────────────────────────────────────────────────────────────────────

1. Update your website

2. Rebuild APK with new version number

3. Distribute new APK or update on Play Store

4. Users will be notified to update
