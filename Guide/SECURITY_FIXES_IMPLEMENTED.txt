## TECHNICAL DETAILS

### Session Timeout Implementation
- Constant: SESSION_TIMEOUT = 1800 seconds (30 minutes)
- Tracks: $_SESSION['last_activity']
- Action: Calls logout() with message on timeout
- Logs: 'SESSION_TIMEOUT' event in audit_log

### Rate Limiting Algorithm
- Window: 15 minutes (900 seconds)
- Threshold: 5 failed attempts
- Scope: Per username OR IP address
- Storage: login_attempts table
- Action: Block with "Try again in X minutes" message

### Session Fingerprinting
- Components: User-Agent + First 3 IP octets
- Algorithm: SHA-256 hash
- Storage: $_SESSION['fingerprint']
- Validation: Every request (in session.php)
- Action: Immediate logout + audit log on mismatch

### Password Strength Validation
- Regex checks for each requirement
- Validates on server-side (validatePasswordStrength)
- Real-time client-side indicator
- Color-coded: red→orange→yellow→green

### Audit Logging
- Table: audit_log (unlimited retention)
- Data: user_id, username, action, IP, user_agent, timestamps
- JSON: old_values, new_values for updates
- Performance: Indexed on user_id, action, created_at
