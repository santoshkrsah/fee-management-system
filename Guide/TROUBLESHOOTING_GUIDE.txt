PROBLEM 1: "Database Connection Failed" Error
═══════════════════════════════════════════════════════════════
CAUSE: Wrong database details in config file

FIX:
  1. Open: public_html/config/database.php
  2. Check these lines match your Hostinger database:
     - DB_NAME (should have u12345_ prefix)
     - DB_USER (should have u12345_ prefix)
     - DB_PASS (exact password you created)
  3. Save and refresh website

═══════════════════════════════════════════════════════════════

PROBLEM 2: Blank White Page / Nothing Shows
═══════════════════════════════════════════════════════════════
CAUSE: PHP error or missing files

FIX:
  1. Check all files uploaded correctly
  2. Especially check: index.php exists in public_html
  3. Check folder permissions: 755
  4. Check file permissions: 644
  5. Look at error logs in hPanel → Error Logs

═══════════════════════════════════════════════════════════════

PROBLEM 3: CSS/Styling Not Working (Plain text only)
═══════════════════════════════════════════════════════════════
CAUSE: Assets folder not uploaded or wrong path

FIX:
  1. Verify assets folder exists in public_html
  2. Check folders inside assets:
     - css
     - js
     - images
  3. Check permissions on assets folder: 755
  4. Clear browser cache (Ctrl + Shift + Delete)
  5. Hard refresh: Ctrl + F5

═══════════════════════════════════════════════════════════════

PROBLEM 4: Can't Login / Invalid Credentials
═══════════════════════════════════════════════════════════════
CAUSE: Admin account not in database

FIX:
  1. Go to phpMyAdmin
  2. Open your database
  3. Click on "admin" table
  4. Check if admin user exists
  5. Default login should be:
     Username: admin
     Password: admin123 (or check the actual hash in database)

═══════════════════════════════════════════════════════════════

PROBLEM 5: Images/Files Not Uploading
═══════════════════════════════════════════════════════════════
CAUSE: Permission or folder missing

FIX:
  1. Create "uploads" folder in public_html if not exists
  2. Set permissions to 777 (temporarily for testing)
  3. If works, change back to 755
  4. Check disk space in hPanel → Home

═══════════════════════════════════════════════════════════════

PROBLEM 6: 404 Error - Page Not Found
═══════════════════════════════════════════════════════════════
CAUSE: Files in wrong location

FIX:
  1. All files should be directly in public_html
     ✅ CORRECT: public_html/index.php
     ❌ WRONG: public_html/fee_management_system/index.php
  2. If files are in subfolder, move them up one level

═══════════════════════════════════════════════════════════════

PROBLEM 7: Website Shows But Students Are Missing
═══════════════════════════════════════════════════════════════
CAUSE: Database not imported

FIX:
  1. Go to phpMyAdmin
  2. Select your database
  3. Check if "students" table exists
  4. Click on "students" table
  5. Should show 50 students
  6. If empty, re-import: database_complete_with_students.sql

═══════════════════════════════════════════════════════════════

PROBLEM 8: Website Very Slow
═══════════════════════════════════════════════════════════════
CAUSE: Hosting plan or optimization

FIX:
  1. Check Hostinger plan (upgrade if needed)
  2. Enable Cloudflare in hPanel
  3. Optimize images (compress large files)
  4. Clear cache

═══════════════════════════════════════════════════════════════

PROBLEM 9: Can't Access /admin Page
═══════════════════════════════════════════════════════════════
CAUSE: Admin folder not uploaded

FIX:
  1. Verify admin folder exists in public_html
  2. Check index.php exists inside admin folder
  3. Check permissions: 755
  4. Try: yourdomain.com/admin/index.php

═══════════════════════════════════════════════════════════════

PROBLEM 10: Changes Not Showing
═══════════════════════════════════════════════════════════════
CAUSE: Browser cache

FIX:
  1. Clear browser cache: Ctrl + Shift + Delete
  2. Hard refresh: Ctrl + F5  (Windows)
  3. Hard refresh: Cmd + Shift + R  (Mac)
  4. Try in Incognito/Private window
  5. Try different browser

STILL NOT WORKING?
═══════════════════════════════════════════════════════════════

Option 1: Hostinger Support
  - Click chat icon in hPanel (bottom right)
  - Available 24/7
  - Tell them: "PHP website not working, need help"

Option 2: Check Error Logs
  - hPanel → Advanced → Error Logs
  - Look at latest errors
  - Google the error message

Option 3: Start Fresh
  - Delete everything in public_html
  - Re-upload all files carefully
  - Re-do database import
  - Update config file again

QUICK HEALTH CHECK
═══════════════════════════════════════════════════════════════

Run this checklist if website stops working:

□ Database connection working? (Check phpMyAdmin)
□ All files uploaded? (Check public_html)
□ Config file correct? (Check database.php)
□ Permissions correct? (755 folders, 644 files)
□ PHP version compatible? (7.4 or higher needed)
□ Disk space available? (Check hPanel)
□ Domain pointing correctly? (Check DNS settings)

═══════════════════════════════════════════════════════════════

FILE STRUCTURE CHECK
═══════════════════════════════════════════════════════════════

Your public_html should look like this:

public_html/
  ├── index.php          ← Main entry file
  ├── admin/             ← Admin panel
  ├── assets/            ← CSS, JS, images
  ├── config/            ← database.php here!
  ├── includes/          ← PHP helper files
  ├── modules/           ← Feature modules
  └── database_complete_with_students.sql (backup)
