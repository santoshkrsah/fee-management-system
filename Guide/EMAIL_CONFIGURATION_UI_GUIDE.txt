## WHAT'S NEW

You can now configure email settings directly from the web interface!
No more manual PHP file editing required.

## HOW TO USE

### Step 1: Access Email Configuration Page

1. Login to your Fee Management System as **sysadmin**
2. Go to: **Settings → Email Configuration**
   OR
   Visit: http://localhost:8000/admin/email_test.php

### Step 2: Configure Email Settings

You'll see a form with all email configuration options:

**SMTP Settings:**
- ✅ Enable/Disable Email Sending (toggle switch)
- ✅ SMTP Host (e.g., smtp.gmail.com)
- ✅ SMTP Port (e.g., 587)
- ✅ Encryption Type (TLS or SSL)

**Authentication:**
- ✅ SMTP Username (your email address)
- ✅ SMTP Password (app-specific password)

**Email Display:**
- ✅ From Email (what recipients see)
- ✅ From Name (your school name)

### Step 3: Save Configuration

1. Fill in all required fields (marked with *)
2. Click **"Save Configuration"** button
3. You'll see a success message when saved
4. Settings are immediately active!

### Step 4: Test Email

1. After saving configuration
2. Scroll down to "Send Test Email" section
3. Enter your email address
4. Click "Send Test Email"
5. Check your inbox (and spam folder)

## SETTINGS API (FOR DEVELOPERS)

### Read Settings
```php
require_once '../includes/settings_helper.php';

// Get single setting
$smtpHost = SettingsHelper::get('smtp_host', 'smtp.gmail.com');

// Get multiple settings
$settings = SettingsHelper::getMultiple(['smtp_host', 'smtp_port']);

// Get all email settings
$emailConfig = SettingsHelper::getEmailSettings();

// Check if configured
$isConfigured = SettingsHelper::isEmailConfigured();
```

### Write Settings
```php
// Set single setting
SettingsHelper::set('smtp_host', 'smtp.gmail.com', 'string', $adminId);

// Set multiple settings
$settings = [
    'smtp_host' => ['value' => 'smtp.gmail.com', 'type' => 'string'],
    'smtp_port' => ['value' => 587, 'type' => 'int']
];
SettingsHelper::setMultiple($settings, $adminId);

// Save email settings
SettingsHelper::saveEmailSettings($configArray, $adminId);
```

## TROUBLESHOOTING

### Problem: Can't access configuration page
**Solution:** Make sure you're logged in as sysadmin role

### Problem: Settings not saving
**Solution:** Check database permissions and error logs

### Problem: Test email fails
**Solution:**
1. Verify SMTP credentials
2. Check firewall/network settings
3. For Gmail: Ensure 2FA enabled and app password generated
4. Check spam folder

### Problem: Email still uses old settings
**Solution:**
1. Make sure you clicked "Save Configuration"
2. Look for success message
3. Refresh the page to see updated values
